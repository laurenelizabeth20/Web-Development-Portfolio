Testing and Debugging

Errors, Exceptions, and Warnings

System error ― there’s a problem with the system or external devices with which the program is interacting.

Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.

User error ― the user has entered data incorrectly, which the program is unable to handle.

Exception: an error that produces a return value that can then be used by the program to deal with the error.

Stack trace: a sequence of functions or method calls that lead to the point where the error ocurred.

Warning: occurs when there is an error in the code that isn't enough to cause the program to crash.

Strict mode: produces more exceptions and warnings and prohibits the use of some deprecated features.

Not using strict mode is referred to as sloppy mode.

Linting tools test the quality of JavaScript code.

The alert method is the most basic form of debugging.

Alert stops a program from running until OK is clicked.

Use the console.log or console.trace to use the console to debug.

Debugger keyword: creates a breakpoint in your code that will pause the execution of the code.

Use the throw statement to throw your own exceptions.

Test-driven development: the process of writing tests before any actual code.

Jest: a testing framework that makes it easy to create and run tests by providing helper methods for common test assertions.
